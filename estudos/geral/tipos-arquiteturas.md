# Tipos de Arquiteturas

## MVP - (Model-View-Presenter) 
  É um padrão de projeto estrutural focado em separar a lógica de negócios da interface do usuário, dividindo o software em três componentes distintos: Model (dados), View (interface) e Presenter (mediador)

## MVC - (Model-View-Controller) 
  É um padrão de design de software que separa a aplicação em três camadas distintas para organizar o código: Model (dados/regras de negócio), View (interface do usuário) e Controller (intermediário/fluxo). Essa separação reduz o acoplamento, facilita a manutenção e permite que a interface seja alterada sem afetar os dados
