# Issues de inception

    Issues (Problemas / Tarefas)
        -São registros de:
            - problemas (bugs),
            - melhorias,
            - novas funcionalidades,
            - tarefas técnicas,
            - dúvidas ou discussões sobre o projeto.
        Pense nelas como um “sistema de chamados” do projeto.
        -Para que servem?
            - Organizar o trabalho
            - Documentar decisões
            - Registrar bugs e ideias
            - Facilitar colaboração
        Mostrar maturidade no desenvolvimento
        Labels (Etiquetas)
        As Issues podem ter labels, como:
            - bug
            - enhancement
            - documentation
            - good first issue
            - help wanted
        Isso ajuda a priorizar e classificar o trabalho.
        Assign (Responsável)
        Você pode atribuir a Issue a alguém (ou a você mesmo).
        Mostra:
            - responsabilidade
            - organização
            - gestão de tarefas
    Milestones (Marcos)
        Milestones são conjuntos de Issues agrupadas em um objetivo maior.
        Pense nelas como:
            - “Entregas”, “versões” ou “fases” do projeto.
            - Para que servem?
            - Planejar versões do sistema
            - Controlar progresso
            - Organizar o roadmap
        Ter visão de curto e médio prazo
    Relação Issue × Milestone
        Uma Issue = tarefa individual
        Uma Milestone = conjunto de tarefas com objetivo comum
